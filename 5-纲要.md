# Chapter 5 网络层：控制平面
## 5.1 导论
### 5.1.1 两种控制平面架构
#### 传统每路由器分布式控制平面
- 每台路由器内置独立路由算法，路由器间交互控制信息
- 分层：控制平面（路由计算）、数据平面（查表转发）

#### SDN集中式控制平面
- 远程集中控制器，各交换机内置本地控制代理CA
- 控制器统一下发流表，控制与数据平面物理分离

### 5.1.2 路由基础概念
#### 路由目标
计算源到目的最优路径，是子网到子网
#### 网络图抽象
- 图模型：$G=(N,E)$
  - $N$：路由器集合；$E$：双向链路集合
  - $c_{A,B}$：链路(A,B)的直接代价，不可达链路代价为$\infty$
#### 汇集树与最优原则
- 汇集树：单节点到全网所有节点的最短路径构成的树
- 路由算法核心：为所有路由器生成汇集树

### 5.1.3 路由选择算法评价准则
1. 正确性：完整覆盖所有目的网段，分组逐跳正确转发
2. 简单性：算法简单、通信开销低
3. 健壮性：拓扑/流量变化快速适配，规避故障链路
4. 稳定性：路由无频繁震荡
5. 公平性：各主机资源分配均衡
6. 最优性：最小化指定代价（允许次优平衡开销）

### 5.1.4 路由算法分类
#### 按信息范围划分
1. 全局算法（LS链路状态）：全网完整拓扑、链路代价
2. 分布式算法（DV距离向量）：仅邻居链路代价，迭代交换路由表
#### 按自适应能力划分
1. 静态：路由表预配置，无法适配网络变化
2. 动态：周期性/事件触发更新，自适应拓扑变更

## 5.2 路由选择算法
### 5.2.1 链路状态LS路由算法
#### 核心工作四步
1. 发现直连邻居，获取邻居地址
2. 测量本地到各邻居链路代价
3. 构造LS分组，记录本地邻居链路开销
4. 全网扩散(泛洪)LS分组，同步全网拓扑信息
#### 泛洪防环机制
- Seq序列号：每个路由器记录是否发送过该分组
- Age生存时间字段；可靠确认重传（类似TTL）
- 有确认
#### Dijkstra最短路径算法（LS核心计算）
**符号定义**
- $c_{i,j}$：i-j链路代价；$D(v)$：源到v当前最小代价
- $p(v)$：v的前一跳节点；$N'$：已确定最短路径永久节点集合

**算法流程**
1. 初始化：源$D=0$，直连邻居$D=c_{source,v}$，其余$D=\infty$
2. 循环：
   1. 选取$N'$外$D$最小节点$w$加入永久集合
   2. 对$w$所有临时邻居$v$更新代价：$D(v)=\min(D(v),D(w)+c_{w,v})$
3. 全部节点加入$N'$后终止，生成转发表

**复杂度**
- 时间复杂度：朴素实现$O(n^2)$，堆优化$O(n\log n)$
- 消息复杂度：$O(N\cdot E)$

#### LS实际协议
OSPF、IS-IS

### 5.2.2 距离向量DV路由算法
#### 核心：Bellman-Ford方程
$$D_x(y)=\min_{v}\{c_{x,v}+D_v(y)\}$$
- $D_x(y)$：x到y最小代价；$c_{x,v}$：x到邻居v代价；$D_v(y)$：v通告的到y代价
#### 算法运行逻辑
1. 周期/链路变更时，向邻居发送本地距离向量（整张路由表）
2. 收到邻居DV后，按BF公式重算所有目标代价
3. 路由表发生变化则重新通告邻居
4. 异步迭代分布式收敛

#### 存在缺陷：计数到无穷
- 现象：故障链路“坏消息”传播极慢，路由代价无限递增
#### 解决方案
水平分割/毒性逆转：不向到达目标的邻居通告该目标路由，向对应邻居通告该目标代价为无穷大
- 防止坏掉时生成环路
- 如：D → C → B → A
  - C向B说到A是无限大，向D说到A的距离是2
  - 如果A坏了
    - 无毒性逆转：B告知C后，C通告自己有CBA路径，导致B以为C有路，把去A的路指向C，形成环路B→C→B
    - 有：B与C确认后，认为A不可达，C与D确认……
#### DV实际协议
RIP

### 5.2.3 LS与DV对比
#### 消息开销
- LS：全网泛单节点链路信息，总消息$O(N\cdot E)$
- DV：仅邻居交互完整路由表，单节点通信量小
#### 收敛速度
- LS：Dijkstra集中计算，收敛更快
- DV：迭代传播，收敛慢，易出现路由环路
#### 健壮性
- LS：错误链路代价仅局部影响
- DV：错误路由表会扩散至全网

## 5.3 AS内部路由（域内IGP）
### 5.3.1 层次路由思想
#### 平面路由缺陷
- 全网路由器维护完整拓扑，规模扩大后存储/计算/传输开销爆炸
- 管理问题：不同网络希望自己管理同时对外隐藏自己的细节
#### 自治系统AS（层次路由）
- AS：统一管理的路由器集合，唯一标识ASN
- 分层路由
  - 域内intra-AS：内路由器运行相同的路由协议
  - 域间inter-AS：运行AS间路由协议
- 1个ISP可包含多个AS
#### 分层路由优势
1. 可扩展性：AS对外视为单个节点，路由表规模可控
2. 管理隔离：各AS可独立选择内部路由协议，隐藏内网拓扑

### 5.3.2 OSPF协议（LS型IGP）
#### 基础特性
1. 开源标准，基于LS+Dijkstra
2. LS分组AS内泛洪，IP报文承载（无UDP/TCP封装）
3. 报文身份认证，防伪造攻击
#### 增强能力
1. 支持等价多路径负载均衡
2. 单播/组播路由一体化
3. 支持层次性OSPF：骨干Area 0+普通区域，LSA仅区域内泛洪
   - 边界路由器：连接其它AS
   - 骨干路由器：仅在骨干区域、运行OSPF路由
   - 区域边界路由：联通骨干和本地

### 5.3.3 其他域内协议
- RIP：经典DV协议，30秒周期交换路由
- EIGRP：Cisco私有DV协议，后开源
- IS-IS：ISO标准LS协议，骨干网常用

## 5.4 AS间路由（域间EGP：BGP）[ISP之间路由]
### 5.4.1 BGP定位
互联网域间标准路径向量协议，连接全球各AS
- 允许子网通告其他网络位置
- 提供2个方法：e外部，i内部
- 基于距离向量DV

### 5.4.2 BGP两类会话
#### eBGP
不同AS网关路由器间会话，交换外部可达网段
#### iBGP
同一AS内部路由器会话，同步外部路由信息

### 5.4.3 BGP路由会话和属性
#### BGP会话
2个BGP路由器(“peers”)在一个半永久的TCP连接上交换BGP报文
#### 属性
1. AS-PATH：路由途经AS序列，检测路由环路
   - 转发需添加自己
2. NEXT-HOP：告诉对方跨AS转发的链路
#### 基于策略的路由
- 通过输入策略过滤
  - 不想通过、已有偏好
  - 经济、政治策略

### 5.4.4 路由通告逻辑
1. eBGP接收外部网段路由
2. iBGP同步至AS全部路由器
3. 向外eBGP通告时追加本AS号至AS-PATH
### 5.4.5 BGP路径选择优先级
1. 本地管理偏好策略
2. AS-PATH长度最短
3. 热土豆路由：路由自己出口代价最小
   
### 5.4.6 热土豆路由
优先选择本AS内距离最近的网关转发外部流量，忽略外部AS跳数
- 就是无论如何都选对自己最短的路径、不管下一路由的远近

### 5.4.7 BGP安全问题与防护
#### 威胁
路由泄漏、BGP劫持，无原生合法性校验
#### 解决方案
RPKI：加密证书验证AS网段宣告权限


### 5.4.8 域内/域间协议核心差异
1. 策略：域内优先性能；域间优先商业/访问策略
2. 规模：域内规模可控；域间需支撑全网海量AS
3. 目标：域内最小时延/开销；域间可控跨域转发

## 5.5 SDN控制平面
### 5.5.1 传统路由局限
基于目的地址转发，难以实现精细化流量工程、多路径负载均衡、流差异化路由
### 5.5.2 SDN四大核心特征
1. 数据/控制平面物理分离
2. 交换机基于流表通用匹配+转发（OpenFlow）
3. 控制平面功能在数据交换设备之外实现
4. 控制器上层可编程网络应用（路由、负载均衡、访问控制）
### 5.5.3 SDN控制器分层架构
1. 网络应用层：抽象
   - 路由、负载均衡业务逻辑
   - API
2. 网络状态管理层：维护全网交换机、链路、主机状态
   - 分布式数据库
3. 通信层：与交换机交互（OpenFlow、SNMP）
### 5.5.4 OpenFlow协议
控制器与交换机的TCP交互协议，下发/更新流表规则
### 5.5.5 SDN面临的挑战
- 控制平面：可信、可靠、性能可扩展性、安全的分布式系统
- 网络、协议满足特殊任务的需求
- 全网可扩展性

## 5.6 ICMP因特网控制报文协议
### 5.6.1 功能定位
网络层控制/差错报文，封装于IP数据报传输，由主机、路由器、网关用于传达网络层控制信息

### 5.6.2 报文结构
- 类型+ 编码，首部8字节
- 附带：原始IP首部+前8字节数据报
  - 刚好是TCP序号+确认号、UDP完整头部
### 5.6.3 核心报文类型
1. Type 0：Echo Reply（ping响应）
2. Type 8：Echo Request（ping请求）
3. Type 3：目的不可达（网络/主机/端口/协议）
4. Type 11：TTL超时（traceroute核心）
5. Type 12：IP首部错误

### 5.6.4 Traceroute实现原理
1. 源发送TTL=1、2、3…递增UDP报文至高位无效端口
2. 途经路由器TTL减至0，回复Type11 ICMP超时报文
3. 目标主机收到报文，回复Type3端口不可达报文，终止探测
### 5.6.5 ICMP现实限制
防火墙常屏蔽ICMP，ping/traceroute路径结果失真；ICMP重定向存在劫持风险

## 5.7 网络管理
### 5.7.1 网络管理五大功能
1. 性能管理：吞吐量、链路利用率监测
2. 故障管理：故障检测、告警、处置
3. 配置管理：设备参数维护
4. 计费/账户管理：资源访问限额、统计
5. 安全管理：网络访问权限控制
### 5.7.2 管理架构
- 管理服务器：集中采集、配置设备
- 被管设备：内置代理Agent，上报状态、接收配置
### 5.7.3 主流管理协议
#### 5.7.3.1 SNMP
传统设备监控协议，基于MIB管理信息库，适用于存量老旧设备
#### 5.7.3.2 NETCONF/YANG
- YANG：网络配置数据建模语言
- NETCONF：标准化设备配置交互协议，新式网络设备主流方案
