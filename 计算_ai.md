# 计算机网络全部可运算公式+原理+计算步骤汇总
按章节分层整理，包含时延、吞吐量、传输窗口、拥塞控制、路由、分片、地址、信道效率等全部可计算模型，每条给出**公式、物理含义、分步计算流程**

## 第1章 计算机网络和因特网（时延/吞吐量/交换）
### 1. 分组交换单跳总传输时延（无排队无处理）
1. 公式：$d_{trans} = \frac{L}{R}$
    - $L$：分组长度(bit)，$R$：链路带宽(bps)
2. 含义：把完整分组全部推向链路所需时间
3. 计算步骤：分组比特总数 ÷ 链路传输速率

### 2. N段链路单分组总存储转发时延
1. 公式：$d = \frac{NL}{R}$
    - $N$：链路条数；$L$分组长度；$R$链路速率
2. 原理：每一段都要完整接收再转发，逐段累加传输时延，忽略传播/处理时延
3. 步骤：分组长度 × 链路数量 ÷ 带宽

### 3. 端到端总时延（无拥塞，忽略排队）
$$d_{end-end} = N(d_{proc} + d_{trans} + d_{prop})$$
- $N$：源到目的链路数；$d_{proc}$单节点处理时延；$d_{trans}$单段传输时延；$d_{prop}$单段传播时延
计算步骤：
1. 算出单段总节点时延：处理+传输+传播
2. 乘以链路总条数N得到全程时延

### 4. 流量强度（排队拥塞判断）
$$\rho=\frac{La}{R}$$
- $a$：每秒到达分组数(pkt/s)；$L$单分组比特；$R$链路带宽
原理：比特到达速率/链路输出速率
- $\rho<1$：排队有限；$\rho≥1$：队列无限堆积、持续丢包
步骤：先算每秒总比特流入$La$，再除以带宽R

### 5. 平均吞吐量
$$Throughput_{avg}=\frac{F}{T}$$
- $F$：文件总比特；$T$完整传输耗时
步骤：文件大小 ÷ 传输总时长

### 6. 瓶颈链路吞吐量
$Throughput = min(R_s, R_c)$
- $R_s$发送端速率，$R_c$接收链路速率
原理：整条链路最小带宽决定整体传输速度

### 7. TDM单时隙速率计算
总链路带宽 ÷ TDM时隙数量
例：1.536Mbps，24时隙 → 1536/24 = 64kbps

## 第3章 运输层（可靠传输窗口、RTT、拥塞控制）
### 1. 停等协议信道利用率
$$U_{sender}=\frac{L/R}{RTT+L/R}$$
- 分子：单分组传输时延；分母：完整一轮（发+往返）总耗时
原理：链路有效工作时间占整个发送周期比例
步骤：
1. 计算传输时延$t_t=L/R$
2. 总周期=RTT+$t_t$
3. 利用率=$t_t / 总周期$

### 2. RTT平滑估计（EWMA）
$$EstimatedRTT = (1-\alpha) \times EstimatedRTT + \alpha \times SampleRTT$$
- $\alpha$推荐0.125；SampleRTT：单次实测往返时延
步骤：
1. 旧估值 × 0.875
2. 新采样RTT × 0.125
3. 两者相加得到新平滑RTT

### 3. RTT抖动偏差DevRTT
$$DevRTT = (1 - \beta) \times DevRTT + \beta \times |SampleRTT - EstimatedRTT|$$
- $\beta=0.25$；取采样值与平滑RTT差值绝对值
步骤：
1. 计算本次RTT与预估差值绝对值
2. 旧偏差×0.75 + 差值×0.25

### 4. TCP超时重传时间
$$TimeoutInterval = EstimatedRTT + 4 \times DevRTT$$
步骤：平滑RTT + 四倍抖动偏差

### 5. TCP发送上限约束公式
$$LastByteSent - LastByteAcked \le Min\{cwnd,rwnd\}$$
- cwnd：拥塞窗口；rwnd：接收窗口
原理：已发送未确认字节不能同时超过拥塞、接收窗口两者最小值

### 6. TCP理论发送速率
$$TCP\ rate \approx \frac{cwnd}{RTT}$$
单位：字节/秒
步骤：当前拥塞窗口字节数 ÷ 往返时延

### 7. 慢启动/拥塞避免窗口更新规则
1. 慢启动（SS，$cwnd < ssthresh$）：每收到一个ACK，$cwnd += MSS$；每个RTT窗口翻倍
2. 拥塞避免（CA，$cwnd ≥ ssthresh$）：每完整RTT，$cwnd += MSS$（线性+1）
3. 超时丢失处理：
    $ssthresh = cwnd / 2$，$cwnd = 1MSS$，退回慢启动
4. 3次冗余ACK（轻微拥塞）：
    $ssthresh = cwnd / 2$，$cwnd = ssthresh + 3MSS$，进入快速恢复

### 8. GBN滑动窗口序号约束
$W≤2^n-1$
- n：序列号比特位数；W发送窗口最大尺寸

### 9. 选择重传SR序号约束
$W \le 2^{n-1}$
防止新旧分组序号混淆

## 第2章 P2P文件分发时间计算
### 1. C/S模式最小分发时间
$$D_{C-S} = \max\left\{\frac{NF}{u_s},\ \frac{F}{d_{min}}\right\}$$
- N客户端数量；F文件比特；$u_s$服务器上传速率；$d_{min}$客户端最小下载速率
计算逻辑取两者最大值：
1. 服务器要上传N份文件耗时$NF/u_s$
2. 最慢客户端下载完整文件耗时$F/d_{min}$

### 2. P2P模式最小分发时间
$$D_{P2P} = \max\left\{\frac{F}{u_s},\ \frac{F}{d_{min}},\ \frac{NF}{u_s+\sum u_i}\right\}$$
$\sum u_i$：所有对等节点上传速率总和
三层约束取最大值：服务器单份上传、最慢客户端下载、全网总上传带宽分发全部文件

## 第4章 网络层IPv4分片计算
### IP分片规则
1. 分片数据部分必须是8字节整数倍（分片偏移单位8B）
2. 每个分片最大载荷 = MTU - IP首部长度（默认首部20B）
3. 最后分片flag=0，其余分片flag=1
计算步骤：
1. 单分片最大数据长度 = MTU - 20，向下对齐8的倍数
2. 原始总数据 ÷ 单分片载荷，向上取整分片数量
3. 每片分片偏移 = 前面所有分片数据总长度 / 8

示例：IP总4000B，MTU=1500，IP头20B
单分片最大数据=1480（1480是8倍数）
分片1：1480B数据，偏移0
分片2：1480B数据，偏移1480/8=185
分片3：4000-2960=1040B，偏移370

总主机数=$2^h$
可用主机 = $2^h - 2$（减去网络地址、广播地址）

2. 2的h次方减2

## 第5章 路由算法（Dijkstra最短路径）
Dijkstra算法计算步骤：
1. 初始化：源节点代价D=0，其余节点D=∞，永久集合N'为空
2. 循环：选取不在N'中D最小节点w加入永久集合
3. 对w所有相邻节点v更新：$D(v)=\min(D(v),D(w)+c_{w,v})$
4. 全部节点加入N'停止，生成最短路径转发表

### Bellman-Ford距离向量公式
$$D_x(y)=\min_{v}\{c_{x,v}+D_v(y)\}$$
x到y的最短距离=所有邻居v的“x到邻居代价 + 邻居到y距离”最小值

## 第6章 链路层MAC信道利用率（CSMA/CD以太网）
$$efficiency =\frac{1}{1+5 t_{prop} / t_{trans}}$$
- $t_{prop}$：最大传播时延；$t_{trans}$：最大帧传输时延
步骤：
1. 计算比值$t_{prop}/t_{trans}$
2. 乘以5加1，整体取倒数得到信道有效利用率

## 补充：各类校验运算
### 1. Internet校验和（UDP/TCP/IP）
运算规则：
1. 数据切分16bit字分组
2. 全部16bit二进制相加，溢出进位回卷加到低位
3. 总和按位取反得到校验和
4. 接收方全部数据+校验和相加结果全1则无错

### 2. CRC循环冗余校验（链路层）
步骤：
1. 发送方：数据后补r位0（r是生成多项式位数-1）
2. 模2除法除以生成多项式二进制
3. 余数作为CRC校验码附加数据尾部
4. 接收方用相同多项式模2除，余数0无差错
